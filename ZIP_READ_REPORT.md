# ex0219_memologin.zip 기반 신규 제작 문서 (기획/설계 초안)

> 목적: 기존 코드 평가가 아니라, **"내가 만들고자 하는 것"을 실제로 구현하기 위한 기준 문서**를 새로 제시한다.

## 1. 만들고자 하는 서비스 정의

### 1.1 서비스 한 줄 정의
- 로그인한 사용자가 메모를 작성/수정/삭제/조회할 수 있는 **개인 메모 웹앱**을 FastAPI로 구현한다.

### 1.2 핵심 사용자 가치
- 빠른 로그인과 간단한 메모 관리.
- 삭제 실수 방지를 위한 소프트 삭제.
- 서버 렌더링(Jinja2) 기반의 가벼운 UI.

### 1.3 MVP 범위 (이번 제작 범위)
- 회원가입
- 로그인/로그아웃
- 메모 목록/작성/수정/삭제
- 인증 사용자만 메모 기능 접근

---

## 2. 기능 명세 (요구사항 중심)

### 2.1 인증
- 회원가입 시 이메일 중복 검증 필수.
- 비밀번호는 bcrypt 해시 저장.
- 로그인 성공 시 세션 발급.
- 로그아웃 시 세션 파기.

### 2.2 메모
- 목록: 최신순 정렬.
- 작성: 제목/내용 필수 입력.
- 수정: 작성자 본인만 가능.
- 삭제: 물리 삭제 금지, `m_status=1`로 소프트 삭제.

### 2.3 권한
- 비로그인 상태에서 메모 API/화면 접근 시 401 또는 로그인 페이지 리다이렉트.
- 메모 수정/삭제는 소유권 검증 후 처리.

---

## 3. 기술 설계 방향

### 3.1 백엔드 구조
- `apis/`: HTTP 라우팅
- `service/`: 비즈니스 로직
- `models/`: SQLAlchemy ORM
- `schemas/`: Pydantic 입력/출력 검증
- `templates/`: 서버 렌더링 HTML

### 3.2 인증 설계
- `SessionMiddleware` 사용.
- 세션에는 최소 정보만 저장(`user_id`, `email` 권장).
- 민감 데이터(해시 포함 사용자 전체 객체)는 세션 저장 금지.

### 3.3 오류 응답 규약
- `400`: 잘못된 입력
- `401`: 인증 필요
- `403`: 권한 없음
- `404`: 리소스 없음
- `409`: 이메일 중복

## 4. 데이터/보안 기준

### 4.1 환경변수 정책
- `.env`는 로컬 전용, 배포본에 절대 포함하지 않는다.
- 저장소에는 `.env.example`만 포함한다.
- 세션 키는 `SESSION_SECRET_KEY`로 통일한다.

### 4.2 배포 산출물 정책
- `.venv`, `__pycache__`, OS 바이너리 파일 제외.
- 의존성은 `requirements.txt` 또는 lock 파일 기반 설치.

### 4.3 보안 체크리스트
- [ ] 비밀번호 해시 저장 확인
- [ ] 세션 고정 공격 방지(로그인 시 세션 재생성)
- [ ] 인증 의존성 누락 엔드포인트 점검
- [ ] 민감정보 로그 출력 금지

## 6. 구현 순서 (실행 플랜)

1. 인증 공통 의존성 작성(`get_current_user`)
2. 회원가입 중복검사 + 로그인/로그아웃 정리
3. 메모 CRUD에 소유권 검증 추가
4. 응답 코드/예외 포맷 표준화
5. `.env.example` 생성 및 패키징 제외 규칙 반영
6. 최소 테스트 작성(인증/메모/권한)

---

## 7. 완료 기준 (Definition of Done)
- 비로그인 사용자는 메모 관련 API 접근 불가.
- 로그인 사용자는 본인 메모만 수정/삭제 가능.
- `.env` 및 `.venv`가 배포본에 포함되지 않음.
- 핵심 시나리오 테스트 통과:
  - 회원가입 성공/중복 실패
  - 로그인 성공/실패
  - 메모 작성/수정/삭제/목록
  - 인증 없는 요청 차단
