# 프로젝트 평가 (2026-04-13)

## 총평
- E(환경) 도메인에 집중한 데이터 파이프라인 실험 저장소로 목적이 명확하고, 생성/처리/다중출처/엑셀 파이프라인이 분리되어 학습 및 PoC에 적합함.
- 스크립트 단위 실행 흐름이 간단하고 README 사용법이 직관적임.

## 강점
1. 생성과 처리 단계의 분리
   - `generate_esg_dummy_data.py`(생성)와 `process_esg_dummy_data.py`(처리)의 관심사 분리가 명확함.
2. 이상치/오류 시나리오 내장
   - 음수, 단위 불일치, 누락값 등의 이상치 주입 로직이 있어 품질 테스트에 유용함.
3. 다중 출처 표준화 베이스라인 제공
   - 정형/반정형/비정형 입력을 규칙 기반으로 통합하여 초기 ingestion 검증에 적합함.
4. 엑셀 기반 샘플 파이프라인 제공
   - 실무에서 자주 쓰는 엑셀 입력 시나리오를 포함해 접근성이 좋음.

## 개선 제안
1. 타입 일관성 강화
   - CSV 출력에서 숫자/문자 혼합 필드(`""`과 float 혼용)가 존재해 후속 처리 시 dtype 안정성이 떨어질 수 있음.
2. 스키마 검증 도입
   - 출력 CSV/JSON에 대해 pydantic 또는 pandera 기반 스키마 검증을 추가하면 회귀 방지에 효과적임.
3. 테스트 자동화
   - 최소한의 단위 테스트(파서/단위변환/배출량 계산)와 E2E 스모크 테스트를 CI에 추가 권장.
4. 로깅/관측성 개선
   - `print` 대신 `logging` 기반 구조화 로그와 에러 코드 체계를 도입하면 운영 전환이 쉬움.
5. 규칙 버전/유효기간 처리 강화
   - 이미 `valid_from`, `valid_to` 필드가 있으므로 처리기에서 기준일 기반 룰 선택 로직을 적용하면 정확성이 높아짐.

## 실행 점검 결과
- 샘플 실행(소량 데이터) 기준 생성/처리/정규화 파이프라인은 정상 완료됨.
